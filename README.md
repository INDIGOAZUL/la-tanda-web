## What Is La Tanda?

La Tanda is a **seven-layer Web3 ecosystem**, not a simple tanda app. Tandas, also known as rotating savings groups or ROSCAs, are one layer of the larger platform.

### The Seven Ecosystem Layers

| # | Layer | Purpose |
|---|---|---|
| 1 | Social Network | Feed, stories, comments, reactions, and reputation generated by real participation. |
| 2 | Digital Tandas | Rotating savings groups with 0% commission, on-chain score, and integrated lending. |
| 3 | Web3 Marketplace | Products, services, bookings, seller score, and payments in lempiras or LTD. |
| 4 | LTD Mining | Five mining tiers, 1-12 LTD/day, 500 LTD/day global cap, based on real activity. |
| 5 | On-Chain Reputation | Portable financial score from 300-850 across platform layers. |
| 6 | La Tanda Chain | Sovereign Cosmos SDK + CometBFT blockchain with fixed LTD supply. |
| 7 | MIA AI | Financial assistant with 16 capabilities using Groq Llama 3.3 70B. |

## La Tanda Chain

La Tanda Chain is a sovereign blockchain built with **Cosmos SDK + CometBFT** for community fintech in Latin America.

- Testnet chain ID: `latanda-testnet-1`
- Token: LTD
- Denom: `ultd`, where 1 LTD = 1,000,000 ultd
- Address prefix: `ltd`
- Block time: about 5 seconds
- Consensus: CometBFT, BFT delegated proof of stake
- Active validators: 13+
- Governance: active, with 2 past proposals
- Planned mainnet: Q1 2027
- Community explorer: [exp.utsa.tech/latanda/staking](https://exp.utsa.tech/latanda/staking)

## Quick Start

### Users

1. Go to [latanda.online](https://latanda.online).
2. Create an account with email or Google Sign-In.
3. Join a tanda, post in the feed, mine LTD, or explore the marketplace.

### Developers

1. API documentation: [latanda.online/docs](https://latanda.online/docs/)
2. Dev portal: [latanda.online/dev-dashboard.html](https://latanda.online/dev-dashboard.html)
3. Authentication: JWT through `/api/auth/login`
4. API surface: 160+ production endpoints

### Validators

1. Read the node guide: [latanda.online/la-tanda-chain-node-guide.md](https://latanda.online/la-tanda-chain-node-guide.md)
2. Run the setup helper:

   ```bash
   wget -q https://latanda.online/chain/node-setup.sh -O node-setup.sh
   chmod +x node-setup.sh
   ./node-setup.sh
   ```

3. Review chain seeds at [latanda.online/chain](https://latanda.online/chain/).
4. Join the incentivized testnet by sending 10 LTD testnet and creating a validator transaction.

## Development Setup

This repository is a static frontend mirror. You can serve it locally without a build step.

### Requirements

- Node.js 18+ or any static file server.
- No local database is required for frontend-only documentation and layout work.
- Do not commit `.env` files or local credentials.

### Serve Locally

```bash
npx serve .
```

Then open the local URL printed by `serve`, usually `http://localhost:3000`.

For a quick smoke test, open:

- `/index.html`
- `/dev-dashboard.html`
- `/marketplace-social.html`
- `/chain/index.html`

## Project Structure

```text
la-tanda-web/
├── *.html                    # Static pages for the ecosystem
├── css/                      # Design tokens, component styles, and page styles
├── js/                       # Source JavaScript modules and page behavior
├── assets/                   # Built assets, images, logos, icons, and hashed JS/CSS
├── chain/                    # La Tanda Chain resources: node setup, genesis, chain page
├── docs -> /var/www/html/docs # Deployed API documentation / Swagger UI symlink
├── .github/                  # Bounty templates, PR gatekeeper, and CI workflows
```

Key files and directories:

- `index.html` — main landing page.

- `chain/index.html` — chain landing page.

- `.github/PULL_REQUEST_TEMPLATE.md` — PR requirements.
- `.github/ban-list.txt` — public ban list checked by contribution rules.

Main pages:

## Contributing

La Tanda uses a three-tier GitHub bounty system.

| Tier | Who Can Claim | Reward |
|---|---|---|
| Tier 0 | Anyone | 10-50 LTD |
| Tier 1 | Contributors with at least 1 previous merge | 50-150 LTD |
| Tier 2 | Contributors with at least 2 previous merges | 150-500 LTD |

Contribution rules:

- Every bounty requires a code-verification answer that proves the contributor read the real source.
- The PR gatekeeper rejects PRs without assignment, accounts younger than 30 days, and users listed in `.github/ban-list.txt`.
- Labels include `tier-0`, `tier-1`, and `tier-2`.
- Use one PR per bounty.

Before opening a PR:

1. Read `.github/PULL_REQUEST_TEMPLATE.md` and `.github/ban-list.txt`.
2. Answer the bounty verification question.
3. Sign commits with a verified GitHub email.
4. Keep the PR scoped to the bounty.
