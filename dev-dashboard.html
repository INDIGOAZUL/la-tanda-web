Todas las peticiones autenticadas requieren el header Authorization con un token JWT válido.

// Obtener token de acceso
POST /api/auth/login
{
  "email": "usuario@ejemplo.com",
  "password": "tu_contraseña"
}

// Respuesta
{
  "success": true,
  "data": {
    "auth_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "eyJhbGciOiJIUzI1NiIs...",
    "expires_in": 86400
  }
}
// Usar token en peticiones
GET /api/user/profile
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

Formato de Respuesta

Todos los endpoints retornan un envelope JSON consistente con la estructura { success, data, meta }.

// POST /api/auth/login — 200 OK
{
  "success": true,
  "data": {
    "auth_token": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "user_id": "usr_abc123",
      "name": "Maria Lopez",
      "email": "maria@ejemplo.com"
    }
  },
  "meta": {
    "version": "4.11",
    "timestamp": "2026-03-01T12:00:00Z"
  }
}
// POST /api/auth/login — 401 Unauthorized
{
  "success": false,
  "data": {
    "error": {
      "code": "INVALID_CREDENTIALS",
      "message": "Credenciales invalidas"
    }
  },
  "meta": {
    "version": "4.11",
    "timestamp": "2026-03-01T12:00:00Z"
  }
}
Campo Tipo Descripcion
success boolean true si la peticion fue exitosa, false si hubo error
data object Datos de respuesta (exito) o objeto error con code y message
meta object Metadatos: version del API, timestamp, paginacion cuando aplica

Tip: Siempre verifica response.success === true antes de acceder a data. Los errores incluyen un code legible por maquina y un message en espanol para mostrar al usuario.

// Manejo de errores recomendado
const res = await fetch(`${API_BASE}/api/wallet/balance`, {
  headers: { 'Authorization': `Bearer ${token}` }
});
const json = await res.json();

if (!json.success) {
  switch (res.status) {
    case 401:
      // Token expirado — redirigir a login
      window.location.href = '/auth-enhanced.html';
      break;
    case 403:
      alert(json.data.error.message); // Mostrar mensaje al usuario
      break;
    case 429:
      // Rate limited — esperar y reintentar
      break;
    default:
      console.error('Error:', json.data.error.message);
  }
}

Seguridad: Los errores 500 nunca exponen detalles internos del servidor. Solo retornan el mensaje generico "Error interno del servidor".

WebSocket — Tiempo Real

Conexion WebSocket para la tombola en vivo. Recibe actualizaciones de countdown, asignacion de turnos y resultados en tiempo real.

Conexion

// URL de conexion (requiere JWT + groupId)
wss://latanda.online/ws/lottery?token=JWT_TOKEN&groupId=GROUP_ID

Autenticacion

JWT en query string ?token=. Verificado con HS256. Conexion cerrada con codigo 4001 si invalido.

Heartbeat

Ping/pong cada 30s. Conexiones sin respuesta se terminan automaticamente.

Rooms

Agrupados por groupId. Solo recibis eventos del grupo al que te conectas.

Servidor → Cliente Eventos

Tipo Descripcion Campos
connected Conexion exitosa al room groupId, roomSize
room_update Alguien entro o salio del room roomSize
countdown_start Inicia countdown de tombola groupName, totalSeconds
countdown_tick Cada segundo del countdown remaining
countdown_complete Countdown llego a cero message
lottery_executing Fase de mezcla de turnos groupName, message
turn_assigned Turno revelado (broadcast al grupo) position, userId, userName, totalPositions
your_turn_assigned Tu turno (solo para vos) position, totalPositions, message
lottery_complete Tombola finalizada results[], totalPositions
lottery_error Error durante ejecucion message

Cliente → Servidor Eventos

ping Keepalive manual; servidor responde con pong Sin campos adicionales

Iniciar Tombola en Vivo

La tombola se inicia con un endpoint HTTP. Los clientes conectados al WebSocket reciben los eventos automaticamente.

POST /api/groups/:groupId/lottery-live Inicia tombola en vivo (coordinador)

Ejemplo Completo

// 1. Conectar al WebSocket de tombola
const token = localStorage.getItem('auth_token');
const groupId = 'mi-grupo-id';
const ws = new WebSocket(
  `wss://latanda.online/ws/lottery?token=${token}&groupId=${groupId}`
);

ws.onopen = () => console.log('Conectado al room');

ws.onmessage = (event) => {
  const msg = JSON.parse(event.data);

  switch (msg.type) {
    case 'connected':
      console.log(`En room con ${msg.roomSize} personas`);
      break;
    case 'countdown_tick':
      updateUI(`Tombola en ${msg.remaining}s...`);
      break;
    case 'turn_assigned':
      showTurn(msg.position, msg.userName);
      break;
    case 'your_turn_assigned':
      alert(msg.message); // "Tu turno asignado es #3 de 23"
      break;
    case 'lottery_complete':
      showResults(msg.results);
      break;
  }
};

ws.onclose = (e) => {
  if (e.code === 4001) console.error('Auth fallida');
};

// 2. Keepalive (opcional, el server hace ping/pong)
setInterval(() => {
  if (ws.readyState === WebSocket.OPEN) {
    ws.send(JSON.stringify({ type: 'ping' }));
  }
}, 25000);

Flujo: Conectar WS → esperar en room → coordinador llama POST /lottery-live → recibir countdown_start → ticks → turn_assigned × N → lottery_complete.

Inicio Rapido — SDK Examples

Login + obtener perfil en tu lenguaje preferido. Todos los ejemplos usan la misma API REST.

// JavaScript — Fetch API
const API = 'https://latanda.online';

// 1. Login
const res = await fetch(`${API}/api/auth/login`, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email: 'tu@email.com', password: 'tu_password' })
});
const { data } = await res.json();
const token = data.auth_token;

// 2. Obtener perfil
const profile = await fetch(`${API}/api/user/profile`, {
  headers: { 'Authorization': `Bearer ${token}` }
}).then(r => r.json());
console.log(profile.data);
# Python — requests
import requests

API = "https://latanda.online"

# 1. Login
res = requests.post(f"{API}/api/auth/login", json={
    "email": "tu@email.com",
    "password": "tu_password"
})
token = res.json()["data"]["auth_token"]

# 2. Obtener perfil
headers = {"Authorization": f"Bearer {token}"}
profile = requests.get(f"{API}/api/user/profile", headers=headers)
print(profile.json()["data"])
// Node.js — axios
const axios = require('axios');
const API = 'https://latanda.online';

// 1. Login
const { data: loginData } = await axios.post(`${API}/api/auth/login`, {
  email: 'tu@email.com',
  password: 'tu_password'
});
const token = loginData.data.auth_token;

// 2. Obtener perfil
const { data: profile } = await axios.get(`${API}/api/user/profile`, {
  headers: { Authorization: `Bearer ${token}` }
});
console.log(profile.data);
// Go — net/http + encoding/json
package main

import (
    "bytes"
    "encoding/json"
    "fmt"
    "net/http"
)

const API = "https://latanda.online"

func main() {
    // 1. Login
    body, _ := json.Marshal(map[string]string{
        "email":    "tu@email.com",
        "password": "tu_password",
    })
    resp, _ := http.Post(API+"/api/auth/login", "application/json", bytes.NewBuffer(body))
    var loginRes map[string]interface{}
    json.NewDecoder(resp.Body).Decode(&loginRes)
    token := loginRes["data"].(map[string]interface{})["auth_token"].(string)

    // 2. Obtener perfil
    req, _ := http.NewRequest("GET", API+"/api/user/profile", nil)
    req.Header.Set("Authorization", "Bearer "+token)
    resp2, _ := http.DefaultClient.Do(req)
    var profile map[string]interface{}
    json.NewDecoder(resp2.Body).Decode(&profile)
    fmt.Println(profile["data"])
}

Headers de Rate Limit

Nginx incluye estos headers en las respuestas cuando aplican limites:

Header Descripcion Ejemplo
Retry-After Segundos para esperar antes de reintentar Retry-After: 1 
// Manejo de 429 con retry
async function fetchWithRetry(url, opts, retries = 2) {
  const res = await fetch(url, opts);
  if (res.status === 429 && retries > 0) {
    const wait = parseInt(res.headers.get('Retry-After') || '1') * 1000;
    await new Promise(r => setTimeout(r, wait));
    return fetchWithRetry(url, opts, retries - 1);
  }
  return res;
}

FAQ — Preguntas Frecuentes

Como obtengo HNL de prueba?

Registrate en latanda.online. Cada cuenta nueva recibe 1,000 HNL de prueba automaticamente en su wallet. Usa POST /api/auth/login con tus credenciales para obtener tu token.

Cual es el formato del token JWT?

El auth_token es un JWT firmado con HS256. Expira en 24 horas. El payload incluye userId, email, y role. Guardalo como auth_token y envialo como Authorization: Bearer <token>.

Como funciona el refresh token?

El login retorna un refresh_token junto al auth_token. Cuando el auth_token expire (401), envia POST /api/auth/refresh con { "refresh_token": "..." } para obtener un nuevo par de tokens sin pedir credenciales al usuario.

Que es LTD y como lo gano?

LTD es el token nativo de La Tanda Chain. Podes ganar LTD contribuyendo al proyecto (ver Bounties), participando en tandas, o recibiendo referrals. El token vive en la blockchain latanda-testnet-1 con denominacion ultd (1 LTD = 1,000,000 ultd).

Hay un SDK oficial?

Aun no. El API es REST puro con JSON, funciona con cualquier HTTP client. Ve los ejemplos en JavaScript, Python, Node.js, Go y cURL. Un SDK oficial de JavaScript esta en desarrollo — contribuciones via bounties son bienvenidas.

Puedo correr un nodo de la cadena?

Si. Descarga el script de setup o sigue la guia en la pagina de la cadena. Necesitas un VPS con Ubuntu 24.04, 4GB RAM, y Go 1.24+. El nodo se sincroniza con el genesis en 168.231.67.201:26656.
