Getting Started

La Tanda es una plataforma fintech y social para Honduras que incluye tandas de ahorro rotativo, marketplace de servicios y productos, sistema de lotería predictiva, y más.

Base URL
Todas las peticiones deben realizarse a:
https://latanda.online
Formato de Respuesta
Todas las respuestas siguen el formato:
{
  "success": true,
  "data": { ... },
  "message": "..."
}
Contenido
Usa el header Content-Type: application/json en todas las peticiones

Autenticación

La Tanda usa tokens JWT (JSON Web Tokens) para la autenticación. Obtén un token mediante el endpoint de login y úsalo en el header Authorization: Bearer <token> para todas las peticiones protegidas.

Flujo de Autenticación

  1. El usuario envía credenciales a POST /api/auth/login
  2. El servidor retorna un JWT en data.token
  3. Incluye el token en el header: Authorization: Bearer <tu-token>
  4. El token se invalida al hacer POST /api/auth/logout o cambiar contraseña
Ejemplo - Login 
// Petición de login
const response = await fetch('https://latanda.online/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    email: 'usuario@ejemplo.com',
    password: 'tu-contraseña'
  })
});

const { success, data } = await response.json();
const token = data.token; // Guarda el token

// Petición autenticada
await fetch('https://latanda.online/api/user/profile', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
Importante
Los tokens son revocados al cerrar sesión o cambiar contraseña. Un token revocado retornará 401 Token ha sido revocado.

Ejemplos de Código

Ejemplos prácticos de las operaciones más comunes usando fetch:

1. Iniciar Sesión

JavaScript 
const login = async (email, password) => {
  const response = await fetch('https://latanda.online/api/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ email, password })
  });
  const { success, data } = await response.json();
  return data.token;
};

const token = await login('usuario@email.com', 'mi-contraseña');

2. Obtener Grupos

JavaScript 
const getGroups = async (token) => {
  const response = await fetch('https://latanda.online/api/groups', {
    headers: { 'Authorization': `Bearer ${token}` }
  });
  const { success, data } = await response.json();
  return data.groups;
};

const groups = await getGroups(token);
console.log('Grupos:', groups);

3. Crear una Publicación

JavaScript 
const createPost = async (token, content) => {
  const response = await fetch('https://latanda.online/api/feed/social/post', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({ content })
  });
  return await response.json();
};

await createPost(token, '¡Hola comunidad de La Tanda! 👋');

4. Usando cURL

cURL 
# Login
curl -X POST https://latanda.online/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"usuario@email.com","password":"pass123"}'

# Obtener perfil
curl -X GET https://latanda.online/api/user/profile \
  -H "Authorization: Bearer TU_TOKEN_AQUI"

# Obtener grupos
curl -X GET https://latanda.online/api/groups \
  -H "Authorization: Bearer TU_TOKEN_AQUI"

Rate Limits

La API tiene límites de velocidad para proteger el servidor. Estos son los límites actuales:

Endpoint / Categoría Límite Descripción
/api/auth/* 5/min Login, registro, reset de contraseña
API General 5/segundo Mayoría de endpoints autenticados
Endpoints públicos 10/segundo Documentación, información pública
Wallet / Transacciones 10/min Depósitos, retiros, transferencias
Social Feed 30/min Publicaciones, comentarios, likes
Headers de Rate Limiting
Cada respuesta incluye headers para ayudarte a controlar el rate limit:
  • X-RateLimit-Limit - Límite total de peticiones
  • X-RateLimit-Remaining - Peticiones restantes
  • X-RateLimit-Reset - Tiempo hasta reset (Unix timestamp)

Manejo de Errores

La API retorna códigos de estado HTTP estándar junto con mensajes de error descriptivos en el cuerpo de la respuesta.

Formato de Respuesta de Error

JSON 
{
  "success": false,
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "El email es requerido",
    "details": { "field": "email" }
  }
}

Códigos de Error Comunes

400
Bad Request - Datos inválidos o faltantes
401
Unauthorized - Token inválido o expirado
403
Forbidden - No tienes permisos
404
Not Found - Recurso no existe
409
Conflict - Conflicto de datos
429
Too Many Requests - Rate limit excedido
500
Internal Server Error - Error del servidor
503
Service Unavailable - Servicio temporalmente no disponible

La Tanda Chain API

La Tanda Chain es la blockchain nativa de la plataforma. Puedes interactuar con ella mediante RPC y endpoints REST:

RPC Endpoint

Conecta tu wallet o aplicación directamente a La Tanda Chain:

https://rpc.latanda.online

API REST Blockchain

Consulta información de bloques, transacciones y tokens:

https://latanda.online/api/chain

Consultar Balance

Obtén el balance de cualquier dirección:

GET /api/chain/balance/:address

Transacciones

Busca transacciones por hash o dirección:

GET /api/chain/tx/:hash
