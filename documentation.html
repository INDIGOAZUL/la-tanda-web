Getting Started

Welcome to the La Tanda Web3 API documentation. La Tanda is a comprehensive fintech and social platform featuring:

  • Rotating savings groups (tandas)
  • Marketplace for services and products
  • Blockchain lottery predictions
  • Wallet management with deposits/withdrawals
  • Social feed and notifications

Base URL: https://latanda.online/api

All endpoints return JSON with {success: boolean, data: {}, message: string}

Authentication

Login

Obtain a JWT token via login, then use it in Authorization: Bearer <token> header for protected routes.

curl -X POST https://latanda.online/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "securepass123"
  }'

Response:

{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user": { ... }
  }
}

Tokens expire (typically 24h) and can be refreshed or revoked on logout/password change.

Code Examples

Get Wallet Balance

curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  https://latanda.online/api/wallet/balance

Get User Profile

curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://latanda.online/api/user/profile

JavaScript Fetch Example

const token = 'YOUR_TOKEN';
fetch('https://latanda.online/api/wallet/balance', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
})
.then(res => res.json())
.then(data => console.log(data));

Rate Limits

  • Login: 5 requests per minute
  • Protected API: 5 requests per second
  • General: 10 requests per second

Exceeded limits return 429 Too Many Requests.

Error Handling

All errors follow this format:

{
  "success": false,
  "error": "Detailed error message"
}

Common errors:

  • 401 "No token provided"
  • 401 "Invalid or expired token"
  • 401 "Token ha sido revocado" (logout or password change)
  • 403 Forbidden (insufficient permissions)
  • 429 Too Many Requests (rate limit)

La Tanda Chain API

Interact with the La Tanda blockchain via:

  • RPC Endpoint: https://rpc.latanda.online
  • REST API: /api/chain/* (e.g., block queries, tx status)
