Welcome to La Tanda API

Authentication

JWT Bearer Token

La Tanda uses JWT (JSON Web Token) for authentication. Include the token in the Authorization header:

Authorization: Bearer YOUR_JWT_TOKEN

Authentication Flow

  1. Login: POST /api/auth/login with credentials
  2. Receive Token: Response includes JWT token
  3. Use Token: Include in Authorization header for all protected endpoints
  4. Refresh: Use /api/auth/refresh to get a new token

Example: Login Request

// Login example
const response = await fetch('https://latanda.online/api/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    email: 'user@example.com',
    password: 'your-password'
  })
});

const data = await response.json();
const token = data.data.token;

// Use token in subsequent requests
const protectedResponse = await fetch('https://latanda.online/api/user/profile', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

Code Examples

Common Operations

1. Login

curl -X POST https://latanda.online/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "your-password"
  }'

2. Get User Groups

const response = await fetch('https://latanda.online/api/groups', {
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN'
  }
});

const groups = await response.json();
console.log(groups.data);

3. Create a Post

const response = await fetch('https://latanda.online/api/feed/social', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    content: 'Hello, La Tanda community!',
    type: 'text'
  })
});

Rate Limits

Rate Limiting Tiers

Endpoint Type Limit Description
Login 5 requests/minute Prevents brute force attacks
API Endpoints 5 requests/second Standard API operations
General 10 requests/second Non-critical endpoints

Rate Limit Headers

X-RateLimit-Limit: 300
X-RateLimit-Remaining: 295
X-RateLimit-Reset: 1609459200

Error Handling

Standard Error Response

{
  "success": false,
  "error": {
    "code": "AUTH_INVALID_TOKEN",
    "message": "Token has been revoked"
  },
  "timestamp": "2026-03-07T13:00:00Z"
}

Common Error Codes

HTTP Status Error Code Description
400 VALIDATION_ERROR Invalid request parameters
401 AUTH_INVALID_TOKEN Token is invalid or revoked
403 FORBIDDEN Insufficient permissions
404 NOT_FOUND Resource not found
429 RATE_LIMIT_EXCEEDED Too many requests
500 INTERNAL_ERROR Server error

La Tanda Chain API

Blockchain Endpoints

La Tanda Chain is built on Cosmos SDK v0.53.6 (CometBFT consensus).

Chain Information

  • Chain ID: latanda-testnet-1
  • Token: LTD (200M fixed supply, 0% inflation)
  • Address Prefix: ltd

RPC Endpoints

# Get chain status
curl https://latanda.online/chain/rpc/status

# Get block information
curl https://latanda.online/chain/rpc/block?height=12345

# Get transaction
curl https://latanda.online/chain/rpc/tx?hash=0x...

REST Endpoints

# Get account balance
curl https://latanda.online/chain/api/bank/balances/ltd1...

# Get validators
curl https://latanda.online/chain/api/staking/validators

# Get chain parameters
curl https://latanda.online/chain/api/params
