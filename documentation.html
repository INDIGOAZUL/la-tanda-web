Autenticación

La API de La Tanda utiliza JWT (JSON Web Tokens) para la autenticación. Todos los endpoints protegidos requieren un token válido en el header Authorization: Bearer <token>.

HTTP Header 
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

Flujo de Autenticación

  1. El usuario se registra o inicia sesión en /api/auth/login
  2. El servidor retorna un access_token (JWT) y un refresh_token
  3. Usa el access_token en todas las requests subsiguientes
  4. Cuando el token expire, usa /api/auth/refresh con el refresh_token
  5. Al cerrar sesión (POST /api/auth/logout), los tokens son revocados
Ejemplo: Login 
POST /api/auth/login
Content-Type: application/json

{
  "email": "usuario@ejemplo.com",
  "password": "contraseña123"
}
Respuesta 
{
  "success": true,
  "data": {
    "user": {
      "user_id": "user_abc123",
      "name": "Juan Pérez",
      "email": "juan@ejemplo.com",
      "role": "user"
    },
    "access_token": "eyJhbGciOiJIUzI1NiIs...",
    "refresh_token": "refresh_token_value",
    "expires_in": 3600
  }
}
Importante:
  • Los tokens son revocados al cerrar sesión o cambiar contraseña
  • Un token revocado retornará 401 Token ha sido revocado
  • Usa el refresh_token antes de que expire para obtener uno nuevo

Ejemplos de Código

Ejemplos prácticos de cómo usar la API de La Tanda en diferentes escenarios.

Iniciar Sesión

cURL 
# Login
curl -X POST https://latanda.online/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "usuario@ejemplo.com",
    "password": "contraseña123"
  }'
JavaScript / Fetch 
const response = await fetch('https://latanda.online/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    email: 'usuario@ejemplo.com',
    password: 'contraseña123'
  })
});

const { access_token } = await response.json();

Obtener Grupos

cURL 
# Get Groups (con JWT)
curl -X GET https://latanda.online/api/groups \
  -H "Authorization: Bearer TU_ACCESS_TOKEN"

Crear un Post

JavaScript / Fetch 
// Create Social Post
const response = await fetch('https://latanda.online/api/feed/social/post', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'Authorization': `Bearer ${access_token}`
  },
  body: JSON.stringify({
    content: '¡Hola La Tanda!',
    visibility: 'public'
  })
});

WebSocket - Lottery Live

JavaScript 
// Connect to Lottery WebSocket
const ws = new WebSocket('wss://latanda.online/ws/lottery');

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('Lottery update:', data);
};

// Authenticate
ws.send(JSON.stringify({
  type: 'auth',
  token: 'TU_ACCESS_TOKEN'
}));

Rate Limits

La API implementa rate limiting para proteger los recursos. Los límites varían según el tipo de endpoint.

Tipo de Endpoint Límite Descripción
/api/auth/* 5/min Login, registro, refresh
/api/* 5/s Endpoints de API generales
Otros 10/s Endpoints públicos
Respuesta cuando se excede el límite 
{
  "success": false,
  "error": "Too many requests. Please try again later.",
  "retry_after": 30
}
Tip:
  • Implementa exponential backoff en tu cliente
  • Cachea respuestas cuando sea posible
  • Usa el header Retry-After para reintentar

Manejo de Errores

La API retorna códigos de estado HTTP estándar junto con un cuerpo JSON que contiene detalles del error.

400
Bad Request
Parámetros inválidos o faltantes
401
Unauthorized
Token inválido, expirado o revocado
403
Forbidden
No tienes permisos para este recurso
404
Not Found
Recurso no encontrado
429
Too Many Requests
Rate limit excedido
500
Internal Server Error
Error interno del servidor

Formato de Error

Error Response 
{
  "success": false,
  "error": "Token ha sido revocado",
  "code": "TOKEN_REVOKED"
}
