La Tanda Developer Documentation

Welcome to the La Tanda platform API. Use this page for quick onboarding and practical examples before switching to the complete OpenAPI reference.

Base URL: https://latanda.onlineAuth: JWT BearerFormat: JSON

Authentication

Authenticate with POST /api/auth/login. The response includes a JWT token. Send it in every protected request:

Authorization: Bearer <your_jwt_token>

API Overview

Main endpoint families:

Auth (/api/auth/*)Wallet (/api/wallet/*)Groups / Tandas (/api/groups/*)Social Feed (/api/social-feed/*)Marketplace (/api/marketplace/*)Lottery (/api/lottery/*)Admin (/api/admin/*)

Code Examples

Login

curl -X POST https://latanda.online/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"dev@example.com","password":"your-password"}'

Get Groups/Tandas

curl https://latanda.online/api/groups \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"

Create a Social Feed Post

const resp = await fetch('https://latanda.online/api/social-feed/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${token}` },
  body: JSON.stringify({ content: 'Hola comunidad La Tanda 👋' })
});
const data = await resp.json();

Error Handling

Typical error response format:

{
  "success": false,
  "error": "Validation failed",
  "code": "VALIDATION_ERROR",
  "details": ["email is required"]
}
  • 400 Invalid payload
  • 401 Missing/invalid token
  • 403 Not enough permissions
  • 404 Resource not found
  • 429 Rate limit exceeded
  • 500 Internal server error

La Tanda Chain API

Use chain endpoints for on-chain visibility and sync status.

  • /api/blockchain/rpc - JSON-RPC bridge for chain methods.
  • /api/blockchain/blocks/latest - Latest block snapshot.
  • /api/blockchain/tx/{hash} - Transaction lookup.
